Systemic Pipeline Issues (traced to source)

1. Window Glazing Holes
Dark facade blocks converted to gray_stained_glass, creating swiss-cheese appearance. Hardcoded DARK_BLOCKS set (lum 25-58) is too aggressive on dark-facade buildings. 30% cap helps but still leaves many holes.
mesh-filter.ts:902-984
voxelize-glb.ts:1929
Affects: chrysler, ansonia, willistower, esb, artinstitute
2. Height Truncation
Google 3D Tiles LOD tiles for upper floors of tall buildings are city-block-scale meshes that can't be isolated. Multi-angle banded capture helps but fundamental LOD limitation means buildings >200m lose 50-80% of height. Setbacks, crowns, and spires are lost.
tiles-headless.ts (capture)
Google 3D Tiles LOD (external)
Affects: esb, chrysler, willistower
3. Neighbor Inclusion
OSM mask dilation includes adjacent structures (plazas, shared-wall buildings). Post-morph re-mask clips edges but can't remove structures within the dilated polygon. Connected structures survive even at dilate=1.
mesh-filter.ts:3477-3616
voxelize-glb.ts:2429-2469
Affects: citigroup, oculus, willistower, artinstitute
4. Column/Pole Artifacts
Street furniture (light poles, traffic signals) within capture radius survive OSM masking because they're not "building" features. isolatePrimaryBuilding() annexRadius=2 keeps poles touching main AABB within 2 blocks.
mesh-filter.ts:5870-5969
mesh-filter.ts:3133-3200
Affects: artinstitute, flatiron, esb
5. Uniform Gray Surface
Zone normalization + wall de-bake (1.5x brightness boost) maps all buildings to the same narrow palette: smooth_stone, light_gray_concrete. GRAY_BLOCKS exclusion set has 6 entries, but the de-bake always pushes toward lighter gray. Every building looks the same shade.
voxelize-glb.ts:1972-2051
Affects: all 10 buildings
6. Satellite Ref Mismatch
Some satZoom levels are too tight (z20 for Art Institute shows only rooftop HVAC equipment) or too loose. No relationship between capture radius and sat zoom — set manually per building.
iterate-grade.ts:330-360
Affects: artinstitute (z20 too tight), citigroup (z20 roof-only)

Overall Verdict

The VLM consistently inflates scores by +1.5 points on average. The VLM is too lenient — it gives partial credit for vague similarities and doesn't penalize enough for lost identity. A building that's "a gray box with a stick on top" (ESB) gets 5.0 from the VLM when it should be 3.

Only 1/10 buildings passes my 7+ bar (Pennzoil). Angular features (45-degree cuts, trapezoids) survive voxelization. Curves (Oculus wings), art deco detail (Chrysler crown), and height-dependent features (ESB setbacks) are all destroyed.

Root causes in priority order:

  1. Height truncation (external) — Google 3D Tiles LOD limitation. No pipeline fix possible. Must choose shorter buildings or accept loss.
  2. Window glazing (fixable) — The DARK_BLOCKS set in mesh-filter.ts:910 is too aggressive. Needs luminance-adaptive threshold or should be disabled for buildings with dark facades. Quick win.
  3. Uniform gray surface (fixable) — Wall de-bake always pushes to lighter gray. Need to preserve captured color variation. The 1.5x brightness boost at voxelize-glb.ts:2043 washes out facade character.
  4. Neighbor inclusion (partially fixable) — OSM mask with dilate=1 still captures connected structures. Need per-building manual polygon or tighter isolation.
  5. Column artifacts (fixable) — Street furniture survives isolation filters. Need to filter by aspect ratio (very tall + very thin = pole, not building).

Building set critique: 5/10 buildings are skyscrapers (ESB, Chrysler, Citigroup, Willis, Pennzoil) where Google 3D Tiles can't capture upper floors. The set should favor shorter, distinctive buildings where the full form can be captured. The Oculus was a poor choice — organic curves don't voxelize. Art Institute was a poor choice — its identity is a ground-level facade that's invisible from standard angles.
